üìã Prerequisites

Before starting, ensure you have the following installed on your system:

1

Visual Studio Code

Latest Version: Download and install from https://code.visualstudio.com/

  • Windows: Download the installer and run it
  • macOS: Download the .zip, extract, and move to Applications
  • Linux: Use your package manager or download the .deb/.rpm file
2

Git

Version 2.30+: Required for cloning repositories

  • Windows: Download Git for Windows
  • macOS: Install via Homebrew: brew install git or download from git-scm.com
  • Linux: sudo apt install git (Ubuntu/Debian) or sudo yum install git (RedHat/CentOS)
Verify Installation: git --version

Node.js & npm

Version 18.0.0+: JavaScript runtime and package manager

  • All Platforms: Download from https://nodejs.org/ (LTS version recommended)
  • Alternative (macOS/Linux): Use nvm for version management

Chrome Browser

Version 90+: Required for the Chrome Extension

  • Download from https://www.google.com/chrome/
  • Microsoft Edge (Chromium-based) also works

‚å®Ô∏è Terminal Setup & Commands

1

Open Integrated Terminal

Press Ctrl+` (backtick) or go to Terminal ‚Üí New Terminal

2

Install Dependencies

Run the following command in the terminal:

npm install

This will download all required Node.js packages (may take 2-3 minutes).

What's happening: npm reads package.json and installs all dependencies into node_modules/
3

Install Playwright Browsers

Workstation uses Playwright for browser automation. Install the required browsers:

npx playwright install --with-deps

This downloads Chromium, Firefox, and WebKit browsers (~300MB).

4

Configure Environment Variables

Create a .env file for configuration:

cp .env.example .env

Edit the .env file and set a secure JWT secret:

# Generate a secure secret
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# Add to .env file
JWT_SECRET=your_generated_secret_here
JWT_EXPIRATION=24h
PORT=3000
NODE_ENV=development
‚ö†Ô∏è CRITICAL: The server will NOT start without a valid JWT_SECRET! This is required for security.
5

Build the Application

Compile TypeScript to JavaScript:

npm run build

This creates the dist/ folder with compiled code.

6

Generate Download Artifacts

Build the Chrome extension and other downloadable packages:

npm run build:downloads

This creates ZIP files in public/downloads/ for easy distribution.

7

Start the Server

Launch the Workstation server:

npm start

You should see output like:

üîß Troubleshooting

Problem: "JWT_SECRET is not defined" error

Symptoms: Server fails to start with authentication error

Solution:
  1. Generate a secret: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
  2. Add to .env file: JWT_SECRET=your_generated_secret
  3. Restart the server: npm start

Problem: "Port 3000 already in use"

Symptoms: Server fails to start, error about port already in use

Solution:
  1. Option 1: Kill the process using port 3000 
    # Windows
netstat -ano | findstr :3000
taskkill /PID [process_id] /F

# Mac/Linux
lsof -ti:3000 | xargs kill -9
  2. Option 2: Change the port in .env file: PORT=3001

Problem: Chrome extension not loading

Symptoms: Extension shows errors or doesn't appear in chrome://extensions/

Solution:
  1. Ensure you built the extension: npm run build:downloads
  2. Check that chrome-extension/manifest.json exists and is valid
  3. Click "Load unpacked" and select the chrome-extension/ folder (not a subfolder)
  4. If errors appear, click "Errors" to see details and fix them
  5. Try removing and re-adding the extension

Problem: npm install fails

Symptoms: Error messages during npm install

Solution:
  1. Clear npm cache: npm cache clean --force
  2. Delete node_modules/ and package-lock.json
  3. Reinstall: npm install
  4. If still failing, check your Node.js version: node --version (must be 18+)

Problem: TypeScript build errors

Symptoms: npm run build fails with TypeScript errors

Solution:
  1. Run linting to see specific issues: npm run lint
  2. Check tsconfig.json is not modified
  3. Ensure all dependencies are installed: npm install
  4. Try cleaning and rebuilding: 
    rm -rf dist/
npm run build

Problem: Dashboard shows "connection refused"

Symptoms: Cannot access dashboard at localhost:3000

Solution:
  1. Verify server is running (check terminal for server output)
  2. Check the port in .env matches the URL you're visiting
  3. Try restarting the server: Stop with Ctrl+C, then npm start
  4. Check firewall settings aren't blocking localhost connections
